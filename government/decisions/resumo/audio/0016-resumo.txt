Resumo da decisão zero zero um seis do AIEOS. Tornamos o repositório do GitHub público, para habilitar a atualização automática dos usuários finais, mas limpamos o conteúdo sensível do histórico antes, para que nada privado fosse exposto. Primeiro, ajustamos a proteção daqui pra frente: o git ignora agora as entradas de memória, então os resumos das conversas ficam apenas locais e nunca vão para o repositório público. Segundo, reescrevemos todos os doze commits para apagar dois arquivos de todo o histórico: a memória desta conversa, e uma decisão de outro projeto, o LeadBroker, que havia entrado por engano. Demos um push forçado do histórico limpo. Terceiro, tornamos o repositório público. Verificamos sem autenticação: a interface do repositório responde duzentos, ou seja, é público; e os dois arquivos sensíveis respondem quatrocentos e quatro, ou seja, sumiram de verdade, não só da ponta. A verificação de atualização, que antes dava erro, agora diz que está atualizado, então a atualização automática dos usuários funciona. A consequência: os usuários agora se atualizam sozinhos; a memória das conversas fica só local, não é mais enviada a lugar nenhum; e os registros de desenvolvimento do AIEOS ficam públicos, o que é normal para um projeto de código aberto. O portão ficou verde com quatorze regras e zero erros. Decisão encerrada.
