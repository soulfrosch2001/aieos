# Design Council

Status: stable
Type: council
Owner: creative-director
Extends: architecture-council

## Purpose
The Design Council adjudicates core system and world-design decisions where systems and
worldbuilding meet — the choices that set how an RPG line plays and holds together. It
decides cross-cutting design direction; it never writes campaigns or prose, never sets
canon facts in isolation (that is the [lore-council](../lore-council/)'s room), and never
owns release scope (that is the [line-producer](../../01-executive/line-producer/)'s).

## Participants
- **Chair** (breaks deadlocks): [creative-director](../../01-executive/creative-director/) — holds the creative/coherence veto.
- **Core** (must attend): [rules-designer](../../03-systems/rules-designer/), [lore-master](../../02-worldbuilding/lore-master/), [campaign-writer](../../04-narrative/campaign-writer/).
- **Advisory** (as needed): [balance-designer](../../03-systems/balance-designer/), [encounter-designer](../../03-systems/encounter-designer/), [quest-designer](../../04-narrative/quest-designer/), [chief-auditor](../../01-executive/chief-auditor/).

## When convened
Convened for T2+ design decisions that span systems and world — see
[engagement-tiers.md](../../../kernel/laws/engagement-tiers.md). Not standing; the
[rpg-orchestrator](../../01-executive/rpg-orchestrator/) convenes it when a decision crosses
department boundaries.

## Inputs
- The design proposal and the problem it solves for players (Directive #1).
- Relevant prior decisions from the [canon](../../memory/registers/canon.md) register.
- Balance and pacing evidence from the [encounter-log](../../memory/registers/encounter-log.md).
- Any affected campaign briefs or quest patterns.
