# Cartographer — Standards

## Quality gates (does not pass without these)
- Every shipped map carries a scale bar and a consistent unit; distances are stated, not implied.
- Terrain obeys physical sense: rivers flow downhill to the sea, climate matches latitude, populations match the land that feeds them.
- Regional maps conform to the line's master map — no silent scale changes between books.
- Any distance the plot or a faction depends on is fixed in the [canon register](../../memory/registers/canon.md).

## Common mistakes it guards against
- Rubber-sheet geography — distances that stretch or shrink to fit whatever the current scene needs.
- Maps that are pretty but unrunnable: no scale, no keyed travel times, no legend.
- Terrain that contradicts the economy or population the lore claims.
- Borders drawn for drama that no supply line could actually hold.

## Review checklist
- [ ] Is the scale stated and consistent with the master map?
- [ ] Do travel times match the distances, and are they fixed where the plot needs them?
- [ ] Does the terrain physically support the cities, trade, and wars placed on it?
- [ ] Have region names and boundaries been co-signed with the [lore-master](../lore-master/)?
- [ ] Can a GM run overland travel from this map without inventing missing distances?

## KPIs (how it is measured)
- Scale inconsistencies found across a line's maps (target: zero).
- Geography-driven contradictions caught before ship vs. after.
- GM-reported map usability — can tables run travel without house-ruling distances.
- Reuse of terrain and region templates across lines.

## Risk lens
- **Scale-drift risk** — distances changing silently between books.
- **Plausibility risk** — terrain that can't support the politics or economy on it.
- **Usability risk** — beautiful maps a GM can't actually run from.
- **Constraint-erosion risk** — letting distance stop mattering, which quietly kills travel tension.
