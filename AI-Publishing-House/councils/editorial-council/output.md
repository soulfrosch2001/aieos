# Editorial Council — Output

## Output format
Produces **editorial-direction minutes** appended to the company's
`meeting-history` register, using the
[report.template.md](../../../templates/report.template.md) shape:
Summary · Consensus · Remaining concerns / dissent · Risks · Alternatives
considered · Recommendation · Implementation plan · Owners & next steps.

The *Alternatives considered* section is load-bearing: each rejected editorial
direction is named with the reason it lost, so the same standards debate is not
re-run. The *Implementation plan* hands the ruling to the editorial floor — it
names owners and the standard to apply, never the edits themselves (Directive
[#2](../../../kernel/laws/prime-directives.md)).

## Dissent
Always present and named. An editor who argued for a different direction — or who
believes the chosen standard harms a title — is recorded by name and reason, so a
failed standard shows who flagged it (Directive
[#8](../../../kernel/laws/prime-directives.md)). "Unanimous" is written only when
true.

## Updates to memory
- Always: [editorial-standards](../../memory/registers/editorial-standards.md) —
  the ruling and recorded dissent (this register extends the stdlib
  [lessons-learned](../../../memory/registers/lessons-learned.md)).
- Always: the company `meeting-history` — the dated minutes.
- If the ruling reshapes the list or a title's identity:
  [catalog-decisions](../../memory/registers/catalog-decisions.md).
- If it changes release expectations: [rights-and-sales](../../memory/registers/rights-and-sales.md).
