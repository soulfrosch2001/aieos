# 02-Development
Status: stable
Type: department
Owner: screenwriter
Extends: none

## Mission
Development is where films begin as ideas and leave as scripts worth producing.
This department is accountable for the studio's story pipeline: generating
concepts, drafting and revising screenplays, pressure-testing structure and
character, and shepherding material to the point where the [greenlight-council](../councils/greenlight-council/)
can make a confident yes/no. We turn a logline into a shootable script, and we
say "not yet" loudly when it is not ready.

## Lead
[screenwriter](screenwriter/) — accountable for the department's output and quality:
the scripts that leave development and the development slate's creative health.

## Agents
| Agent | Role in one line |
|-------|------------------|
| [screenwriter](screenwriter/) | Writes and revises the screenplays; owns the words on the page. |
| [story-editor](story-editor/) | Diagnoses structure, character, and theme; the script's toughest reader. |
| [development-executive](development-executive/) | Curates the slate and packages projects toward greenlight. |

## Councils it sits on
- [greenlight-council](../councils/greenlight-council/) — brings scripts forward for a make/don't-make decision.
- [creative-council](../councils/creative-council/) — defends story coherence against the rest of the studio.

## Memory it feeds
- [project-ideas](../memory/registers/project-ideas.md) — concepts, loglines, and abandoned threads worth keeping.
- [greenlight-decisions](../memory/registers/greenlight-decisions.md) — what was greenlit and the case made for it.
- [production-log](../memory/registers/production-log.md) — development lessons handed downstream.

## Lifecycle
Every agent in this department follows the same agent contract and lifecycle.
A department adds agents by copying [../../templates/agent.template.md](../../templates/agent.template.md).
Work is sized to an [engagement tier](../../kernel/laws/engagement-tiers.md); the
[studio-orchestrator](../01-executive/studio-orchestrator/) routes and fans out,
never directing the story (Directive [#2](../../kernel/laws/prime-directives.md)).
