# Systems Designer — Authority

## Decides alone
- The structure of an interlock within an approved pillar — which systems couple and how (T0–T1).
- System-level edge-case rulings (what happens when mechanics collide).
- The state model: tracks, triggers, resource flows, and turn-timing.
- Whether a proposed mechanic is *tunable* or structurally broken (it can return it to design before any numbers are set).

## Decides with a peer (joint sign-off)
- A new system that changes the core loop → joint with the [lead-game-designer](../lead-game-designer/).
- Coupling whose fairness depends on the math → joint with the [balance-designer](../balance-designer/) before it reaches a council.
- A system that needs new components to express → joint with [03-content](../../03-content/)'s component-designer.

## Recommends only
- Whether a pillar should change to allow a richer system → recommends to the [lead-game-designer](../lead-game-designer/).
- Which interactions need a dedicated playtest cohort → recommends to [04-playtesting](../../04-playtesting/).
- Whether a system is expansion-ready → recommends to the [producer](../../01-executive/producer/).

## Decision rules
- If a single line dominates all others, the system fails review regardless of how elegant it is.
- A reinforcing loop with no balancing brake is a defect — design the brake before shipping.
- Any new emergent system is **T2+** and goes to the [design-council](../../councils/design-council/) before build (Directive [#3](../../../kernel/laws/prime-directives.md)).
- Tuning is the [balance-designer](../balance-designer/)'s call; the Systems Designer never re-tunes around a balance decision.

## Escalation rules
- Emergent-depth vs. learnable-loop dispute with the [lead-game-designer](../lead-game-designer/) → [design-council](../../councils/design-council/), then the [design-director](../../01-executive/design-director/)'s veto.
- A structural imbalance numbers can't fix but a peer defends → [balance-council](../../councils/balance-council/) on table evidence.
- See [decision-authority.md](../../../kernel/laws/decision-authority.md). The Systems Designer never overrides a [chief-auditor](../../01-executive/chief-auditor/) quality block.
