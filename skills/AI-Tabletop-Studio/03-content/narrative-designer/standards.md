# Narrative Designer — Standards

## Quality gates (does not pass without these)
- Every core mechanic has a one-line thematic justification that a player would accept.
- The canon bible has zero internal contradictions across base and expansions.
- No flavor text changes or obscures the rule it accompanies (clears the rules-writer's clarity gate).
- Tone is consistent across all components — no genre whiplash.

## Common mistakes it guards against
- Theme bolted on after the system, contradicting what the player actually does.
- Flavor that bloats a card and crowds out function.
- Inconsistent names for the same world element across components.
- Lore so dense it becomes homework instead of atmosphere.

## Review checklist
- [ ] Each mechanic's theme justification written and sane.
- [ ] Canon bible contradiction pass clean.
- [ ] Flavor fits the word budget set with [rules-writer](../rules-writer/).
- [ ] Art and fiction agree, signed with [component-designer](../component-designer/).
- [ ] Tone consistent across base and current expansion.

## KPIs (how it is measured)
- Theme-resonance signal in [playtest-feedback](../../memory/registers/playtest-feedback.md) ("I felt like X").
- Canon contradictions found post-launch (target: zero).
- Flavor edits forced by clarity gate (lower is healthier alignment).

## Risk lens
- Coherence risk — fiction and mechanics drifting apart.
- Bloat risk — flavor degrading usability.
- Continuity risk — an expansion breaking established canon.
