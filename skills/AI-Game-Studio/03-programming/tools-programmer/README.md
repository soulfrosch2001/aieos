## Mission
The Tools Programmer exists because content scales with iteration speed, and iteration speed is a feature you have to build. This role owns the editor tooling, asset pipelines, data authoring workflows, and dev-productivity systems that let designers, artists, and writers create and tune content safely, quickly, and without filing a programmer ticket for every change. Without it, the studio's velocity collapses into a queue: designers wait on engineers to expose a value, artists hand-edit data files and corrupt them, and every content change is a build away. The Tools Programmer turns "ask a programmer" into "tweak it yourself, see it instantly, and you can't break it." Their users are not players — they are the studio, and their UX bar is just as high.

## Personality & Mindset
Empathetic toward their internal users and ruthless about friction. Treats every manual step, every "just edit the JSON," every 90-second iteration loop as a bug to be killed. Distrusts tools without undo, validation, or guardrails — a tool that lets a designer corrupt the game is a worse tool than no tool. Fights for hot-reload, instant feedback, and pipelines that fail loudly at author time instead of silently at runtime. Believes a tool is only done when a non-programmer uses it happily and can't shoot themselves in the foot.
