This studio inherits the kernel's authoritative
[shared glossary](../../shared/glossary.md) for cross-company governance terms (Prime
Directive, Tier, Orchestrator, Council, Approval/Quality gate, Veto, Disjoint ownership,
Memory register, Recorded dissent), which supersedes the previously-duplicated local
definitions. When a doc uses one of those terms, it means exactly what the shared glossary says.

## Local additions (stricter only)
Game-dev craft vocabulary the shared glossary does not cover:

- **Core loop** — the repeating moment-to-moment cycle of actions that defines the play experience.
- **Design pillar** — a top-level promise the whole game is built to keep (e.g. "every death is fair").
- **Game feel / juice** — the tactile responsiveness of input, feedback, and animation.
- **Frame budget** — the per-frame time allowance (e.g. 16.6 ms at 60 FPS) every system must fit inside.
- **Economy** — the in-game system of resources, sinks, and faucets.
- **Balance** — tuning so no option is dominant or useless; tracked in [balancing-history](../10-memory/).
- **Replayability** · **Accessibility** · **Playtest** · **Vertical slice** — degree of repeat reward; inclusive design; structured observation of real play; a polished cross-section proving the experience.
- **Release candidate (RC)** · **Hotfix / patch** · **Technical debt** · **Telemetry** — ship-gated build; targeted live fix; deferred cleanup logged in [10-memory/](../10-memory/); captured gameplay data.
