# Proofreader — Standards

## Quality gates (does not pass without these)
- A complete final pass on the actual typeset proof — not the manuscript — is on record.
- No unresolved critical defect remains: no dropped or duplicated lines, no wrong figures, no broken cross-references.
- Running heads, page numbers, TOC, and index verified against the actual pages.
- Every correction round that changed pagination triggered a fresh targeted pass before lock.
- The required tier gates are met (Directive #9) before print-readiness is signed.

## Common mistakes it guards against
- Proofreading the manuscript instead of the typeset pages, missing layout-introduced defects.
- Re-editing at proof stage — making substantive changes that belong to copy or developmental.
- Trusting a corrected proof without re-checking the reflow the correction caused.
- Skipping the TOC/index/cross-reference verification under deadline pressure.
- Letting the lock date override an unresolved critical defect.

## Review checklist
- [ ] Pass made on the typeset proof, page by page.
- [ ] No literals, bad breaks, widows/orphans, dropped or duplicated lines remain.
- [ ] Running heads, folios, captions correct on every page.
- [ ] TOC, index, and cross-references match actual pages.
- [ ] Each correction round re-checked for reflow before lock.
- [ ] Marked proof handed to the book-designer in clean, unambiguous notation.

## KPIs (how it is measured)
- Post-print errata count — the truest measure of what the proof stage missed (lower is better).
- Critical-defect escape rate to the locked file.
- Number of correction rounds to a clean proof.
- Reflow-introduced error rate — errors created by applying corrections and not re-caught.

## Risk lens
- Catastrophe risk — a critical error reaching print, the most expensive failure in the pipeline.
- Layout risk — typesetting silently breaking correct text.
- Reflow risk — corrections introducing new errors on re-pagination.
- Schedule-pressure risk — the lock date forcing a pass to be cut short.
