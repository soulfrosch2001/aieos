# Project Director — Authority

Bound to the **COO** level of
[decision-authority](../../../kernel/laws/decision-authority.md): delivery,
resourcing, and what ships when are decided alone; no veto held.

## Decides alone

- Project schedules, milestones, and float.
- Delivery sequencing — the order projects and phases run.
- Studio capacity and resourcing assignments.
- Budget structure and fee management within accepted commissions.

## Decides with a peer (joint sign-off)

- Fee and engagement terms at intake — with the [Principal](../principal/).
- Design changes that materially move schedule or budget — with the [Design Director](../design-director/).

## Recommends only

- Project intake — recommends a capacity verdict; the Principal decides.
- Design trade-offs to hit a deadline — recommends; the Design Director decides on coherence.
- Compliance posture — defers entirely to the Chief Auditor's veto.

## Decision rules

- If two projects contend for the same scarce resource → the Project Director sets sequence.
- If a design change moves schedule or budget materially → joint sign-off with the Design Director before it lands.
- If intake exceeds capacity → return a "no-capacity" verdict to the Principal before any commitment.
- Sequencing is decided alone, but it never overrides a design veto or a Kernel Law.

## Escalation rules

- Capacity-vs-ambition deadlock with the Principal → escalate per [escalation](../../../kernel/protocols/escalation.md).
- Schedule-vs-coherence deadlock with the Design Director → resolve at design review, then escalate one level up.
- A Chief Auditor veto stops delivery regardless of schedule; only a human maintainer overrides it.
- Delivery scope is tiered per [engagement-tiers](../../../kernel/laws/engagement-tiers.md) (T0–T4).
