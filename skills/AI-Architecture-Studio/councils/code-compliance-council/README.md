# Code Compliance Council
Status: stable
Type: council
Owner: chief-auditor
Extends: security-council

## Purpose
Decides whether a project meets building code and regulatory requirements before it goes to permit and before field work is accepted. It is the studio's compliance gate — life-safety, accessibility, zoning, energy code, and permit-readiness. It does **not** judge design quality (the [design-review-council](../design-review-council/)'s domain) and it does **not** set schedule or budget (the [project-director](../../01-executive/project-director/)'s domain). It exists to keep the firm from building something it cannot legally occupy.

## Participants
- **Chair** (breaks deadlocks): [chief-auditor](../../01-executive/chief-auditor/) — holds the compliance veto; never designs, never directs.
- **Core** (must attend): [project-architect](../../04-delivery/project-architect/), [structural-engineer](../../03-technical/structural-engineer/), [building-systems-engineer](../../03-technical/building-systems-engineer/).
- **Advisory** (as needed): [construction-administrator](../../04-delivery/construction-administrator/), [sustainability-designer](../../02-design/sustainability-designer/), [bim-specialist](../../03-technical/bim-specialist/), [lead-architect](../../02-design/lead-architect/).

## When convened
Before a permit submission and before acceptance of compliance-relevant field work, sized by tier — see [engagement-tiers.md](../../../kernel/laws/engagement-tiers.md). **T2** gets a compliance sign-off; **T3** strategic or high-risk work needs council plus executive. The [studio-orchestrator](../../01-executive/studio-orchestrator/) convenes; the council is not standing.

## Inputs
- The permit set and applicable code analysis.
- Engineering proof from [03-technical](../../03-technical/) and the field record from delivery.
- Prior findings in [code-compliance-log](../../memory/registers/code-compliance-log.md) and relevant [design-decisions](../../memory/registers/design-decisions.md).
