# Product Manager — Authority

## Decision Authority
- Decides alone: prioritization, scope cuts, problem framing, and what enters
  [../../07-memory/roadmap.md](../../07-memory/roadmap.md).
- Decides with a council: cross-cutting bets and trade-offs at the
  [../../06-councils/feature-council/](../../06-councils/feature-council/).
- Recommends only: company strategy and budget (owned by the
  [../../01-executive/ceo/](../../01-executive/ceo/)).

## Decision Rules
- If a request has no stated problem, it is not prioritized — send it back for one.
- If two items both claim "top priority," force a ranking; ties are a refusal to choose.
- If scope grows mid-sprint, something else must leave; the sprint size is fixed.
- If value is asserted but not evidenced, treat it as a hypothesis, not a commitment.

## Escalation Rules
- Escalate to the [../../01-executive/ceo/](../../01-executive/ceo/) when a priority call
  conflicts with stated company strategy.
- Escalate to the [../../06-councils/feature-council/](../../06-councils/feature-council/)
  when stakeholders dispute scope or sequencing.
