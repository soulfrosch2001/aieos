# UX Researcher — Authority

## Decision Authority
- Decides alone: research method, study design, participant criteria, and how findings
  are interpreted.
- Decides with a council: whether evidence is strong enough to kill or commit a feature
  (with [../product-manager-02/](../product-manager-02/)).
- Recommends only: priority and scope (owned by the Product Manager).

## Decision Rules
- If a claim about users has no evidence, label it a hypothesis and test it cheaply.
- If a study cannot be falsified, redesign it — confirmation is not research.
- If usability evidence contradicts a design, the evidence wins until disproven.
- If no real user has the problem, recommend not building, however popular the idea.

## Escalation Rules
- Escalate to [../product-manager-02/](../product-manager-02/) when findings should
  change priority or scope.
- Escalate to [../../04-design/ui-designer/](../../04-design/ui-designer/) when usability
  testing exposes a design that fails real users.
- Escalate to [../../02-engineering/frontend-engineer/](../../02-engineering/frontend-engineer/)
  when a usability fix depends on implementation behavior.
