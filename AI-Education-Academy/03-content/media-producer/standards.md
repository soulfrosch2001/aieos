# Media Producer — Standards

## Quality gates (does not pass without these)
- Every asset maps to a brief and a learning objective — no decorative media.
- Source accessibility met: alt text drafted, captions/transcripts on audio and video, color-independent encoding, sufficient contrast.
- One clear focal point and visual hierarchy per frame; legible at the target display size.
- Audio is clean and intelligible; video timing supports comprehension, not flash.

## Common mistakes it guards against
- Redundant media that mirrors prose and adds cognitive load.
- Charts that only work in color or rely on contrast a screen reader cannot relay.
- Overdesigned frames with no focal point — the eye does not know where to land.
- One-off assets that ignore the library and fracture the visual system.

## Review checklist
- [ ] Asset ties to a brief and objective; it teaches what prose cannot.
- [ ] Passes grayscale and screen-reader checks; alt text, captions, transcripts present.
- [ ] Single focal point, clear hierarchy, legible at target size.
- [ ] Audio/video clarity and timing reviewed.
- [ ] Consistent with the visual system and named/versioned for reuse.
- [ ] Accessibility and consistency co-reviewed with the [content-editor](../content-editor/).

## KPIs (how it is measured)
- Asset comprehension lift — does the lesson teach better with the asset than without (from [learning-outcomes](../../memory/registers/learning-outcomes.md)).
- Accessibility conformance rate across produced assets (target 100%).
- Reuse rate from the library versus rebuilt-from-scratch.

## Risk lens
- Cognitive-load risk: media that adds noise instead of clarity.
- Accessibility risk: assets some learners cannot perceive.
- Coherence risk: a fractured visual system that makes the catalog feel inconsistent.
- Production-debt risk: unversioned, unfindable assets that cannot be maintained as content changes.
