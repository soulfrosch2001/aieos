# Agent Template (Folder Layout)

> An agent is a **folder** of small files (each ≤50 lines). Copy this layout to create one.
> See [../00-company/file-structure.md](../00-company/file-structure.md).

```
<agent-name>/
  README.md
  responsibilities.md
  authority.md
  standards.md
  craft.md
```

## README.md
- **Identity** — role, department, reports to, file path. Note `-02` suffix if applicable.
- **Mission** — one paragraph: why this role exists; what's lost without it.
- **Personality & Mindset** — 3–5 sentences, distinct voice, willing to disagree.
- **Index** — links to the four sibling files below.

## responsibilities.md
- **Responsibilities** — what this agent owns end-to-end.
- **Questions This Agent Always Asks** — the 5–10 it raises before agreeing.

## authority.md
- **Decision Authority** — decides alone / with a council / recommends only.
- **Decision Rules** — "if X then Y" heuristics.
- **Escalation Rules** — when it stops and escalates, to whom.

## standards.md
- **Common Mistakes It Guards Against** · **Review Checklist** · **Best Practices** ·
  **Quality Gates** · **Risk Analysis Lens**.

## craft.md
- **Communication Style** · **Collaborates With** (cross-links) · **Updates To Memory**.
- Plus **Coding Philosophy** (engineering) OR **Business Thinking** + **User Thinking**
  (product/design).
