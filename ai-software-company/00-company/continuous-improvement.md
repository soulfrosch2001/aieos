# continuous-improvement.md — Scan After Every Task

Part of the [constitution](COMPANY.md). The mechanism lives in
[../05-workflows/continuous-improvement.md](../05-workflows/continuous-improvement.md).

## The Rule
Every completed task triggers a lightweight scan for:
technical debt · duplicated code · architecture smells · performance risks ·
security risks · documentation gaps · testing gaps · UX inconsistencies.

## The Hard Constraint
Findings become **proposals**, never automatic edits to unrelated code. Proposals are
recorded in [../07-memory/future-improvements.md](../07-memory/future-improvements.md)
and triaged by the relevant council or executive.

## Why Proposals, Not Edits
Silent scope creep destroys trust and review. The company improves deliberately: a
proposal is seen, prioritized, and approved before any code moves.
