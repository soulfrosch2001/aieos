# 07-memory — Corporate Memory

The company's long-term voice. This is where decisions, lessons, debt, and plans
outlive any single task or conversation. Memory is **append-mostly**: we add new
entries and dated corrections, but we never rewrite history or delete dissent.

**Every important decision updates memory.** A change that ships without a memory
update is incomplete. See [../00-company/conventions.md](../00-company/conventions.md)
(Memory Discipline) and the templates in [../09-templates/](../09-templates/).

## The Nine Registers
1. [roadmap.md](roadmap.md) — planned work & priorities; owned by
   [product-manager-02](../03-product/product-manager-02/).
2. [technical-debt.md](technical-debt.md) — debt register: id, owner, severity,
   interest, ADR link.
3. [architecture-decisions.md](architecture-decisions.md) — the immutable ADR log.
4. [meeting-history.md](meeting-history.md) — index of council minutes.
5. [lessons-learned.md](lessons-learned.md) — what went right/wrong and the lesson.
6. [known-issues.md](known-issues.md) — open bugs/risks with severity & workaround.
7. [standards.md](standards.md) — engineering standards the company holds.
8. [coding-rules.md](coding-rules.md) — concrete style, error-handling, testing rules.
9. [future-improvements.md](future-improvements.md) — the proposals queue.

## How To Use
Each register states its purpose, its columns, one example entry, and a "How to
append" note. Append at the bottom; correct by adding, not editing. Keep every
file ≤50 lines — split before it grows.
