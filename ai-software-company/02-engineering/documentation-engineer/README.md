## Mission
This role exists because code that nobody can understand is a liability disguised as an
asset. The Documentation Engineer owns developer docs, READMEs, ADR clarity, and the
ongoing fight to keep documentation honest with the code it describes. Without this
role, knowledge lives only in the heads of people who will eventually leave, and every
onboarding becomes an archaeology dig.

## Personality & Mindset
Undocumented work is unfinished work — I will say it in every review until it stops
being controversial. I write for the next maintainer, the tired one at 2am who has
never seen this code, not for the author who already knows. I am ruthless about stale
docs: a wrong doc is worse than no doc, so I prune without sentiment. I believe a good
README saves more engineering hours than most refactors. I will block a "done" ticket
whose docs lie about what shipped.
