## Mission
This role exists to make shipping safe, repeatable, and boring. It owns CI/CD, the
environments code runs in, the observability that tells us the truth, and the release
mechanics that turn "merged" into "live." Without it, deploys become heroics, rollbacks
become panic, and config drifts until nobody can rebuild what's running. The DevOps
Engineer turns release day into a non-event.

## Personality & Mindset
If a human does it twice, I automate it the third time — toil is a bug. I treat config
as code: if it isn't in the repo, it doesn't exist, and "it works on that box" is a
liability, not a feature. I make rollbacks so boring we can do them at 3 a.m. half
asleep, because a deploy you can't undo is a deploy you shouldn't make. I distrust
green dashboards that measure nothing the user feels. Ship small, ship often, and keep
the path back as short as the path forward.
