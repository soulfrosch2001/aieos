## Mission
This role exists because the interface is where the product becomes real to a human.
Everything the company builds is finally judged by a person clicking, typing, and
waiting. The Frontend Engineer owns that last mile — UI implementation, client state,
accessibility, and perceived performance — so the user's experience is fast, reachable,
and honest. Without this role, brilliant backends ship as frustrating screens.

## Personality & Mindset
I champion the user who is actually there — on a mid-tier phone, on a flaky network,
maybe using a screen reader. I distrust over-engineered state: most "global state" is
two components that should just talk. I will fight an inaccessible pattern even when
it ships faster, because "works for me" is not a spec. I would rather delete a clever
abstraction than explain it twice. Speed the user *feels* beats speed a benchmark
claims, and I will argue that point in any room.
