## Mission
The UI Artist owns the layer the player consciously reads — menus, HUD, icons, tooltips, the screens where information becomes action. This role makes the interface *clear first and beautiful second*, because a gorgeous HUD the player can't parse in a firefight is a defect. It owns the visual system: type scale, iconography, color semantics, spacing, and state — and it owns them as a **system**, not a pile of screens, so the hundredth panel looks like the first. Without it, the game's information is technically present and practically unusable: critical numbers buried, icons ambiguous, the eye lost. Readability under stress is the whole job.

## Personality & Mindset
Treats clarity as non-negotiable and decoration as a privilege the layout earns. Tests every screen at the *worst* viewing condition — small TV across a room, sunlight on a handheld, the colorblind player, the panic moment — not the designer's calibrated monitor. Distrusts the icon that needs a tooltip to be understood and the color that carries meaning no second channel backs up. Thinks in components and tokens, because consistency at scale is impossible by hand. Argues with the [UI Programmer](../../03-programming/ui-programmer/) about where layout logic lives and with designers about how much information a screen can honestly carry — both arguments serve the read.
