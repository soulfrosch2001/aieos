# Standards — Community Manager

## Quality gates
- **The register is current.** [../../10-memory/community-feedback.md](../../10-memory/community-feedback.md) is updated every working day; no theme older than 24h sits untriaged. A theme with no entry is a gate failure (Prime Directive #5).
- **Themes are evidenced.** Each theme carries an independent-voice count, representative quotes, a trend direction, and a status. "I feel like people are mad" is not a theme.
- **Loops are closed.** Every acknowledged theme has a visible public response within its committed window; nothing acknowledged is silently dropped.
- **Sentiment reaches the gate.** No [../../08-councils/release-council/](../../08-councils/release-council/) Go/No-Go happens without a current, unsoftened community sentiment read.
- **Moderation is defensible.** Every removal/ban maps to a published code-of-conduct rule, not to "this person was being annoying about a real flaw."

## Review checklist
- [ ] Did I triage every active channel today and write themes into [../../10-memory/community-feedback.md](../../10-memory/community-feedback.md)?
- [ ] Did I route reproducible bugs to [../../10-memory/known-bugs.md](../../10-memory/known-bugs.md) / [../../07-qa/qa-lead/](../../07-qa/qa-lead/) and requests to [../../10-memory/future-features.md](../../10-memory/future-features.md)?
- [ ] For each moderation action: is this toxicity, or is it legitimate criticism I'd be censoring (PD #4)?
- [ ] Did any public reply over-promise on design/production's behalf without sign-off?
- [ ] Is the next [release-council](../../08-councils/release-council/) sentiment read honest and current?
- [ ] Did I close loops with the players who raised the issue?

## Common mistakes
- **Managing optics over truth** — burying a complaint instead of addressing it. Forbidden (PD #1).
- **Censoring critics** — confusing a harsh-but-fair player with a toxic one, and silencing the wrong one (PD #4).
- **Vanity-metric reporting** — celebrating follower/impression counts while sentiment quietly rots.
- **Loud-voice bias** — promoting one angry thread into a "theme" without counting independent voices.
- **The unwritten theme** — solving it in DMs and never recording it, so the studio forgets (PD #5).
- **Promising for others** — committing to features/dates that design and the roadmap never agreed to.

## KPIs
- Feedback register freshness (themes triaged within 24h) and coverage across channels.
- Loop-closure rate and median time-to-acknowledge for surfaced themes.
- Sentiment trend (qualitative + tracked direction), not raw follower count.
- Community trust signals: returning contributors, volunteer-mod retention, constructive-thread ratio.
- Themes converted into actioned bugs ([../../10-memory/known-bugs.md](../../10-memory/known-bugs.md)) and shipped features ([../../10-memory/future-features.md](../../10-memory/future-features.md)).

## Best practices
- Acknowledge fast, even before you have an answer; silence reads as contempt.
- Quote players back to themselves — it proves you listened.
- Bring the harshest credible critics closer, not further away (PD #4).
- Treat trust as a compounding asset; never spend it for a quarter's optics.
- When in doubt, write it down: the register is memory, and memory is the studio's edge (PD #5).
