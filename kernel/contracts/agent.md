# Contract: Agent

An Agent is the atomic actor of AIEOS: one role, one voice, one set of
responsibilities. Extends [entity.md](entity.md).

## Structure — a folder of exactly 5 files
| File | Defines |
|------|---------|
| `README.md` | Identity, mission, personality/mindset, index of the other files. |
| `responsibilities.md` | What it owns, what it explicitly does **not** own, the questions it always asks. |
| `authority.md` | Decides alone / with a peer / recommends only; decision rules; escalation. |
| `craft.md` | Communication style, working philosophy, key collaborators (with tensions), which memory registers it feeds. |
| `standards.md` | Quality gates, common mistakes it guards against, review checklist, KPIs, risk lens. |

## Required sections
The 5 files map onto the universal agent dimensions every role must define:
**Mission · Responsibilities · Authority · Communication Rules · Escalation Rules
· Decision Rules · Review Checklist · Quality Gates · KPIs · Mindset · Best
Practices · Common Mistakes · Long-Term Responsibilities.**

No agent may omit a dimension; a dimension that is empty for a role is stated as
"none" with a reason. This is what prevents "completely custom" agents — every
agent is the same template, filled differently.

## Rules
- An agent **belongs to exactly one department** (or to the Government).
- Authority must reconcile with [../laws/decision-authority.md](../laws/decision-authority.md).
- An agent **does not implement outside its responsibilities**; it escalates.
- Build a new agent from [../../templates/agent.template.md](../../templates/agent.template.md).

## Conformance
Valid when: 5 files present, identity block complete, every dimension addressed,
all cross-links resolve, authority consistent with the company's decision map.
